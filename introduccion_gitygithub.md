## 🧠 ¿Qué es el control de versiones?

Sistema que permite:
- Guardar versiones del código
- Volver atrás si algo falla
- Trabajar en equipo

💡 Es como el historial de Word, pero para programadores

---

## ❌ Problemas sin control de versiones

- Perdida de código
- Archivos duplicados
- No saber quién hizo cambios
- Conflictos en equipo

---

## 📜 Un poco de historia

- Antes: SVN, CVS
- 2005: Linus Torvalds crea Git

🎯 Objetivo: velocidad + trabajo distribuido

## ⚙️ ¿Qué es Git?

- Sistema de control de versiones
- Funciona en tu computadora
- Guarda historial del proyecto

## ☁️ ¿Qué es GitHub?

- Plataforma en la nube
- Permite compartir código

## 🔄 Flujo básico de Git

1. git init
2. git add .
3. git commit -m "mensaje"

---

## 🧰 Comandos básicos

```
git init
git status
git add .
git commit -m "mensaje"
git log
```

👉 ¿Qué hace cada uno?
- **git init** → crea el repositorio
- **git status** → muestra el estado de los archivos
- **git add** → prepara cambios para guardar
- **git commit** → guarda una versión
- **git log** → muestra el historial

---

## ✍️ Buenas prácticas de commit

❌ Malos mensajes:
- "cambios"
- "arreglos"

✅ Buenos mensajes:
- "Agrega función de login"
- "Corrige error en validación de usuario"
- "Refactoriza clase Usuario"

💡 Tips:
- Usar verbos en presente ("Agrega", "Corrige")
- Ser claro y específico
- Un commit = una idea
